3.2 技术选型详解

层级技术选型选型理由
App端 Flutter + Dart iOS/Android 一套代码，性能接近原生，开发效率高
Web端 Next.js + React SSR友好，SEO强，生态丰富，开发速度快
API网关 Kong / Envoy 高性能，插件丰富，WAF + 限流一体化
后端语言 Go（业务）+ Python（AI） Go高并发、易部署；Python AI生态成熟
Web框架 Gin（Go）+ FastAPI（Python） 轻量、高性能、自动文档
主数据库 TiDB（MySQL兼容） 水平扩展、MySQL协议兼容、Cloud-native
缓存 Redis Cluster 成熟稳定，集群模式高可用
向量数据库 Milvus / Qdrant AI Embedding 相似度检索，开源可控
LLM 开源部署（Llama3/Qwen）+ 微调 成本可控，数据安全，支持私有化部署
对话引擎 vLLM / SGLang 高吞吐 LLM serving，P95延迟低
搜索引擎 Elasticsearch 全文检索、聚合分析、实时搜索
消息队列 Apache Kafka 高吞吐、事件驱动、持久化可靠
CDN 阿里云 CDN + 边缘节点 就近访问、降低延迟、节省带宽
监控 Prometheus + Grafana 可观测性，告警，仪表盘
容器编排 Kubernetes（RKE2/EKS） 生产级容器编排，自愈能力强

五、安全与合规

5.1 安全架构

  • 🛡️ 应用安全：WAF（Web应用防火墙）+ API签名认证 + 限流防刷
  • 🔐 数据安全：传输加密（TLS 1.3）+ 存储加密（AES-256）+ 密钥管理（KMS）
  • 👤 身份安全：手机号实名 + 活体认证 + 行为风控
  • 📊 隐私合规：GDPR + 《个人信息保护法》+ 数据脱敏
  • 🔍 安全审计：渗透测试 + 漏洞扫描 + SOC2 认证（长期）

5.2 合规要求

法规要求实施方案
网络安全法用户实名制、数据本地化手机号认证，数据存国内
个人信息保护法知情同意、最小必要隐私协议、权限申请
算法推荐管理规定算法透明、可关闭推荐解释、一键关闭
