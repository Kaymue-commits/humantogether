二、数据库分库分表策略
🗄️ 分库策略
users 库 → 按 user_id % N
opportunities 库 → 按 category hash
messages 库 → 按 room_id hash
ai_memory 库 → 按 user_id hash
transactions 库 → 按 user_id
⚡ 缓存策略
L1: 本地缓存 5min（Caffeine）
L2: Redis Cluster（热数据）
L3: TiDB（冷数据 + 历史）
热点 Key 分散：tag%N 路由
缓存预热 + 降级策略
🔄 读写分离
主库（写）→ 4个从库（读）
读请求 80% 路由到从库
MSR（MySQL Shell）数据校验
半同步复制（Semi-sync）
PingCAP TiDB-operator
